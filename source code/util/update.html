Changed

Artifact list

  • Added an action to download artifacts via radial menu.
    Seems to download the artifact with option (Merged Configured and Default Values)
  • Added a Trace action to IFlows.
    Only on single environment, Cloud Integration tenants for now, because I don't have access to an Edge Integration Cell to test the differences.
  • Added a Compare action group to Value Mappings, Message Mappings and IFlows.
    Extremely rudimentary at the moment because it's basically a text comparison.
    • You can compare Runtime vs. Designtime, Runtime vs. Runtime on another stage and Designtime vs. Designtime on another stage.
    • Value Mappings are at least decently structured.
    • For IFlows and Message Mappings it's really just a textual comparison of the IFlow/Mapping.
      IFlows are also missing a comparison for scripts/mappings.
    • Shows how many differences there are, if any.
  • Mouse mappings are fixed now, unlike the last two times I thought they were.
    Unfortunately this means that you will most likely have to change them to the buttons you configured previously.
  • The previous Copy Link action was replaced by Details, now displays meta info about the artifact.
    The specific information depends on the artifact type, but generally you will see stuff like:
    • Names
    • IDs
    • Deployment status
    • Deployment version
    • Link to the artifact (You can still copy the link that way)
    • Endpoints
    • Log level
    • Direct dependencies
    • Used adapters
    You can click to copy any piece of information that has a ðŸ“„ symbol next to it.

API Proxy list

  • I don't remember what was available before, but these are the action groups that are available now:
    • Copy Name / Copy ID
    • Open / Open in New Tab / Show Details
    • Stage Switch (Might not work, depending on the naming between stages)
  • Maybe API Providers, Products and Applications will be added in the future.

Appearance

  • All artifact lists have been visually updated to be more user-friendly (indentation etc.) and blend better into the Integration Suite UI.
  • Side navigation and header bar can now be coloured the same color as the Tenant Tag.
    The tag will automatically be set to the default color, when the header is coloured.

Miscellaneous

  • Some settings will now immediately take effect after saving.
    • Artifact lists will be updated (Mainly affects colors and reloading the shortcut list).
    • Side navigation and header bar background color will be updated, but not reset to default without reloading.
    • Fixed list at the bottom will also be updated immediately.
  • Most features should now be disabled, if the configured tenant-based permissions prohibit them
  • Artifacts lists load independently of each other
  • Artifact lists are periodically refreshed. Toggling a list root element will refresh the list, but it's not really reliable yet.
  • You should only have to dismiss the "Camel Version Update" notification once more per tenant (when/if it's running on Camel 3.14)
    Sorry for the annoying repetitive notifications over the past weeks.
  • You can also turn on/off notifications about Integration Suite version updates, if you'd like (Settings -> General -> Reminders -> Version Updates)
    Disabling this will also disable Camel Update notifications.

Added

Secure Material list

  • Has only 2 action groups for now
    • Copy Name / Copy User / Show Details
    • Delete (Careful! You have to confirm it, but it will delete the credential).

Custom Shortcuts

  • This will contain all your custom shortcuts, which you can edit in the overrides tab of the settings.
    I know it's manually editing JSON again, but adding this to the UI settings is going to take some time, probably.
    There are also some things that just aren't supposed to be edited, but I still wanted to leave open a backdoor to do so.
  • A guide on how to create and customize shortcuts can be found by clicking on the table button next to the save button in the settings.
    Some useful demo shortcuts should automatically be created if you:
    • newly install the extension
    • automatically update from the old "official" version
    • reset your settings

Removed

  • The Unlock buttons from anywhere but the artifact list were remove.
  • Documentation options were deleted for now. I couldn't shake the feeling, that it caused serious issues with message mappings for some reason.

Older (Main)

  • Artifact lists will now also load if loading the API Proxies fails.
  • Artifact search based on locks is now available.
    • :lock
      Searches for all artifacts with locks.
    • :lock-me
      Shows only the locks of the logged-in user.
    • :lock-search term
      Shows only the locks of users matching the search term.
  • Search for an artifact/package by pattern
    Case-insensitive. Spaces, underscores, hyphens, parentheses, and square brackets are ignored in the artifact/package names and IDs.
    These should also be omitted in the RegEx.
    • :rx-search term
    • :regex-search term
